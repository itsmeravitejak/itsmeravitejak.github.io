After a lots of trials and errors i finally got fixed into the jekyll world of blogging .Its crazy to blog with jekyll ,actually to explain this whole thing you basically need to have the basical knowledge of hosting and some experience in programming.Before the usage of jekyll i used wordpress a content management system in php,it was a wonderfull experience using wordpress, but the reason to move to jekyll was the markup and its programmer friendly nature.

For example we can post code snippets like this

 #include<stdio.h>
 int main(int argc,char argv[])
 {
  int a=0;
  int *b=a;
  printf("a value:%d",a);
  printf("pointer b value:%d",*b);
 }

Tables are posted like this

Entity      | Result
------------|----------
`&copy;`    |  &copy;
`&pound;`   |  &pound;
`&lambda;`  |  &lambda;
`&mdash;`   |  &mdash;

then it will be converted like this

EntityResult
&copy;©
&pound;£
&lambda;λ
&mdash;—

headings like this

# first level heading
## second level heading
### third level heading 
### fourth level heading

will be converted like this

first level heading

second level heading

third level heading

fourth level heading

easy huh thats why i opted to run this blog with jekyll actually jekyll is just a parser it just does nothing but parses files with tags and markup like the ones i mentioned above into html that you see here
